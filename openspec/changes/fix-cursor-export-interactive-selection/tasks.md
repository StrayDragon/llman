## 1. Correctness（必须修复）
- [ ] 1.1 构建统一的可选择列表：每一项携带稳定标识符（`composer_id` / `tab_id` 或 fallback key），展示与导出共享同一引用列表。
  - 验证：选择任意两项后导出，导出内容与选择项一一对应（不受排序影响）。
- [ ] 1.2 搜索返回稳定标识符：搜索选择不得返回“搜索结果 index”，必须返回同一标识符空间的 key，并与主选择合并去重。
  - 验证：通过“搜索更多”选择项并导出，结果与搜索选择一致。
- [ ] 1.3 去歧义：展示 label 包含短 ID 后缀，避免同标题/同时间误选。
  - 验证：构造重复标题对话，用户仍可精确选择目标项。

## 2. Performance（避免全量加载）
- [ ] 2.1 延迟加载：仅在用户最终确认选择后，按选中的 key 拉取完整对话内容；未选中项不得加载 bubbles。
  - 验证：在大量对话的 fixture 中，仅选少数项时，数据库读取/解析量显著下降（可通过 debug 日志或统计计数验证）。

## 3. Tests（回归覆盖）
- [ ] 3.1 回归测试：sorted summaries 与导出列表顺序不同仍能正确映射（使用 key 而非 index）。
- [ ] 3.2 回归测试：search 选择映射（search 结果 → key → 导出对象）正确。
- [ ] 3.3（可选）性能测试：选择 1 项不触发全量 bubbles 拉取（可用 mock/计数器）。

## 4. Acceptance（验收标准）
- [ ] 4.1 交互导出不再出现“选 A 导出 B”的错配。
- [ ] 4.2 搜索选择与主列表选择可混用且不冲突、不重复。
- [ ] 4.3 大库下选择少量项时不会做全量加载。
